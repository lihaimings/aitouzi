# 量化底座深度优化路线（A股/ETF）

> 目标：把项目从“可运行Demo”升级为“可研究、可验证、可风控、可扩展”的量化投资底座。

## 一、设计原则（来自主流量化实践）

1. **先可验证，再追收益**
   - 必须有清晰的 in-sample / out-of-sample / walk-forward 验证链路。
2. **先约束风险，再放大收益**
   - 关注最大回撤、波动、换手、容量，而不只看收益。
3. **先工程化，再智能化**
   - 先把数据质量、回测一致性、报告可复现做扎实，再叠加Agent或LLM。

## 二、当前项目已具备能力

- 数据：AkShare / Baostock / TuShare（三源）
- 策略：ETF轮动（20/60动量，周频调仓）
- 风控：
  - 单资产权重上限
  - 交易成本（手续费+滑点）
  - 换手上限（max_turnover）
  - 风险平价权重（inverse-vol）
- 验证：Walk-Forward滚动验证
- 报告：
  - Markdown日报
  - QuantStats HTML
  - 飞书推送占位接口

## 三、下一阶段（优先级从高到低）

### P0（必须）
1. 数据质量层 ✅
   - 增加数据完整性校验（缺失率、停牌天数、异常跳变）
   - 输出数据质量报告
2. 回测严谨性 ✅（部分完成）
   - 多基准对比（510300、510500）
   - 参数稳定性热力图（top_n/min_score/vol_lookback）
3. 风险预算 ✅
   - 增加组合波动目标（vol targeting）
   - 增加回撤保护开关（drawdown kill-switch）

### P1（推荐）
1. 组合层升级
   - 接入 PyPortfolioOpt（最小方差/风险预算）
   - 与现有风险平价对照
2. 策略集成
   - 动量 + 均线 + 波动过滤三策略集成投票
3. 报告升级
   - 自动生成策略对照报告（主策略 vs 基准 vs Backtesting.py基线）

### P2（扩展）
1. 实盘桥接准备
   - 统一订单对象（signal -> order -> fill）
   - 失败重试、审计日志
2. Agent/LLM 插件层
   - 仅输出结构化分数（-1~1）
   - 严禁直接下单

## 四、指标体系（建议作为周度追踪）

- 收益与风险：年化收益、波动、Sharpe、Sortino、Calmar、最大回撤
- 稳定性：滚动Sharpe、月度胜率、回撤恢复天数
- 可执行性：换手率、交易成本占比、持仓集中度
- 对照：相对基准超额收益（alpha）、信息比率

## 五、与开源生态关系（避免重复造轮子）

- Backtesting.py：用于基准与策略原型验证
- quantstats：用于报告与风险可视化
- Qlib：中长期升级为多因子+ML研究平台
- PyPortfolioOpt：用于组合优化

结论：当前项目采用“轻量核心 + 开源组件”是合理路径，后续继续向模块化、可插拔演进。
