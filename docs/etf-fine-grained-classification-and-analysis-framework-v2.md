# ETF 细分类标准 v2 + 分类分析框架（可直接用于量化研究）

> 目标：解决“ETF不应一刀切分析”的问题。
> 
> 适用：你当前主线（仅ETF、单边趋势为收益引擎、人民币散户资金）。

---

## 1. 分类原则（从“名称关键词”升级为“研究可用分类”）

采用 **五层标签体系**，每只ETF可多标签：

1) 资产大类（Asset Class）
- Equity（权益）
- Bond（债券）
- Commodity（商品）
- Money（货币现金）
- REITs

2) 地域暴露（Geo Exposure）
- CN_A（A股）
- HK（港股）
- US（美股）
- Global（全球）
- QDII（跨境）

3) 策略属性（Strategy Type）
- Broad Beta（宽基）
- Sector/Theme（行业主题）
- Style Factor（价值/成长/红利/低波等）
- Enhanced/Active（增强）

4) 行业/主题映射（Industry/Theme）
- 周期：有色、煤炭、钢铁、化工、机械、航运等
- 成长：半导体、AI、算力、新能源、军工等
- 消费：食品饮料、可选消费、家电等
- 金融地产：银行、券商、保险、地产
- 防守：公用事业、红利、低波、医药（部分阶段）

5) 风险驱动类型（Risk Driver Regime）
- Cycle Supply-Demand（供需周期驱动）
- Policy-Liquidity（政策流动性驱动）
- Earnings-Valuation（业绩估值驱动）
- Rate-Sensitive（利率敏感）
- FX-External（汇率海外驱动）

## 2. 分类到分析：不同ETF用不同模板

## A) 周期型ETF（供需驱动）
典型：有色、煤炭、钢铁、化工、航运、部分能源链

核心看点：
- 供给端：产能、开工率、库存、停产/检修、地缘扰动
- 需求端：基建、地产、制造业、出口、全球工业周期
- 价格链：现货/期货价差、利润传导、上下游剪刀差

量化特征建议：
- 趋势因子权重提高（中短动量）
- 回撤保护阈值更严格
- 信号更新频率可略高于普通宽基

## B) 成长型ETF（景气+估值驱动）
典型：半导体、AI、软件、机器人、新能源

核心看点：
- 产业景气与订单
- 政策和资本开支周期
- 估值分位与风险偏好

量化特征建议：
- 趋势+波动双因子
- 严格“反打脸机制”（确认期、冷却期）
- 高波动期自动降杠杆/降仓

## C) 防守型ETF（现金流与稳定性驱动）
典型：红利、低波、公用事业、部分债券ETF

核心看点：
- 股息率/现金流稳定性
- 利率与风险偏好切换
- 组合回撤缓冲效果

量化特征建议：
- 作为防守仓分配逻辑的一部分
- 不以高收益为第一目标，以回撤控制和稳定为主

## D) 债券ETF（利率与信用驱动）
核心看点：
- 利率方向（长端/短端）
- 信用利差与风险偏好
- 流动性环境

量化特征建议：
- 熊市或高波动阶段提高配置
- 与权益仓形成风险对冲

## E) 跨境ETF（海外宏观驱动）
核心看点：
- 海外利率、美元指数、汇率
- 海外科技周期与风险事件

量化特征建议：
- 设置独立风险预算上限
- 做分散，不做重仓押注

## 3. 适配你当前系统的落地结构（建议）

你当前项目建议采用“3池+1规则引擎”：

1) Core池（宽基/高流动性）
- 作用：稳定Beta与趋势主引擎

2) Alpha池（行业主题：周期+成长）
- 作用：增强收益弹性

3) Defense池（债券+红利/低波+少量黄金）
- 作用：控制回撤

4) Regime规则引擎
- bull：Core+Alpha提升权重
- bear：Defense权重上升
- choppy：降低换手、提高阈值

---

## 4. 关键提醒（防止策略误用）

1) 同是ETF，不同类别不能共用同一阈值。
2) 周期ETF和成长ETF的风控阈值要分开。
3) 防守ETF不能用“追求最高收益”的评价标准。
4) 所有分类结论最终要在回测+滚动验证中被证实。
